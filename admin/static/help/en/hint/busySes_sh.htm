Top Busy Session

On this page, you will see information of active top busy sessions running on the monitored database server*. They are listed in descending order of the column OP+. The OP+ is the incremental number of operations, such as read/write and networking etc., that are exercised by a session in a refreshing interval, and the SEQ+ is the incremental number of sequential scans exercised by the session in the refreshing interval.

Each session is displayed with how long it has been alive, how many operations have been executed up to now or in a refreshing interval, how many sequential scans have been exercised up to now or in a refreshing interval, and what the database is being accessed etc.

You can click a column head to sort the list in ascending/descending order of the selected column and click or anytime to refresh information on the page.
 

Symptom and Approach

1.

If a session has tremendous amount of operations in OP+ or sequential scans in SEQ+, you may need to drill down to check what SQL is running, how the SQL is running and how the resource is being taken by the SQL.

Related drilling-down options in the right-click menu : Active SQL Tree Diagram, Active SQL in a Session, Start Analysis Session
2. If the busiest session is an abnormal one, you can terminate it with Terminate Session option in the right-click menu. This option will be shown only if you have permission to terminate a session.
