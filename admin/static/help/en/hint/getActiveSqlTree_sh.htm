Active SQL Tree Diagram

This page exhibits the SQL execution tree* and the profile information of the current running SQL in the specified session. You can click or any time to view the execution trees and the profiles of the SQLs running in the seesion one by one, and. If all SQLs in the session have been completed, the message indicating that the session is completed will be shown. If you use to refresh information automatically, you can stop it any time by clicking it again.

Each SQL operator node* displayed on the SQL tree is exhibited with its current phase and estimated cost, the number of rows returned, table used in the operator node, and the action executed by the node.

The profile information presents the running SQL statement, in which session and on which database that the SQL is executed, the estimated cost that the SQL execution will take and the estimated rows that the SQL will return etc.
 

Symptom and Approach

1.

If sequential scan is used in executing the SQL and/or any operator node has unusual high cost, you may need to analyze the session or evaluate the SQL through Start Analysis Session, SQL Evaluator or Execution History of SQL Pattern.
2. If a SQL returns a vast number of rows, application developer may need to check how the SQL statement is written and whether it is necessary to return such vast amount of data to avoid defective application design affecting database performance.
