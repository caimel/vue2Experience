SQL Tree Diagram

This functionality is provided on Informix 11 and above only that supports SQL Trace facility. You have to setup SQL TRACE configuration parameter to be able to use this functionality. Please refer to IBM® Informix® Administrator's Guide for how to configure SQL TRACE.

As SQL Trace has been set, this page will exhibit the SQL execution tree* and related statistics of the traced SQL statement in a session. The information of the oldest traced SQL will be clean out from the SQL Trace. You may enlarge the SQL TRACE setting to keep more traced SQLs before the oldest one is clean out.

Each SQL operator node* displayed on the SQL tree is exhibited with its estimated cost, the number of rows returned, table used in the operator node, and the action had been executed by the node along with the related statistics.

If the traced SQL is clean out from the SQL Trace, the following message is shown instead. You may enlarge the SQL TRACE setting to keep more traced SQLs before the oldest one is clean out. If SQL Trace doesn't setup, you see the same message but the value of the SQL ID is zero.
 

Symptom and Approach

If the average execution time of the SQL is high and/or the cost is high, you may scrutinize the statistics such as I/O, lock and data sort provided on the page and the execution tree to get more information for further diagnosis and analysis.
