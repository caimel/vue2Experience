Show Idle Session

Idle connections are often seen in applications that require user interaction. Even though the user does not send any request to the database engine, the connection remains opened and consumes resources. A possible outcome is that the number of connections reaches to the maximum limit and the database server rejects the connection requests from then on. If you notice the number of connections surging above a normal value, then you may want to check the list of idle connections and terminate idle connections that have been sitting idle for long durations in order to release resources.

You can click a column head to sort the list in ascending/descending order of the selected column and click or anytime to refresh information on the page.


Column Description
Session Identification of a session
User Name of user initiating the session
Idle Time The lapse of time that the connection has been idle
Lock Hold The number of locks held by the session
Lock Wait The number of lock waits experienced by the session
Buffer Read The number of buffer read operations of the session
Buffer Write The number of buffer write operations of the session
Database Name of the database connected by the session
