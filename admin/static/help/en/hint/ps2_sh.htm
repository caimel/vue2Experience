Top OS Process List

On this page, you will see information of system loading and what the top busiest system processes are. The top busiest system processes are listed in descending order of the column %CPU. From the COMMAND, you can tell whether a top busiest process is a database process, "oninit".

The table shows the top busiest system processes. For each process, you are able to know who runs the process, how much time the process uses, what the current state of the process and how much percentage of CPU is occupied by the process etc.

Different OS will provide different process information. The table shown below is an example for your reference.

You can click a column head to sort the list in ascending/descending order of the selected column and click or anytime to refresh information on the page.
 

Symptom and Approach

1.

If the busiest process is a database process, oninit, and its CPU% is unreasonable high, you may need to check the Top I/O Session or Top Busy Session to find out which session is running in the process, how the resource has been taken by the session, and what SQLs are executing etc to get the root cause.
2. If any process that you would like to view doesn't show in the table, you can drill down into the Process List via right-click context menu. It will list all active system processes.
