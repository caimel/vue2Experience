Table Extent List

Both Table Extent List (include system tables) and Table Extent List (exclude system tables) display table/index extent information in the monitored database server*, but the former one includes extent information of system tables.

Each table/index is displayed with its dbspace, database, the number of rows, the size of each row, the allocated and used size of data storage, the number of extents, whether it's a user-defined temporary table, sort file or permanent system temporary table, whether a page lock has been set for it, and its creation date.

You can click a column head to sort the list in ascending/descending order of the selected column and click or anytime to refresh information on the page.

The number of extents and contiguity of extents are important to performance. A large number of extents or discontiguous pages of data cause the database server to spend extra time finding the data. Monitoring table/index extent will help you determine if any reorganizing or altering has to be applied to the table/index or dbspace.

 

Symptom and Approach

1.

You may drill down into Table Related SQLs from the table with a large number of extents to check how SQLactivity* accesses the selected table. If the table is contributing to any frequently-used and heavy-cost SQL activity*, you may need to tune the SQL and/or table schema, and/or change the first and next extent size of the table according to its row size and the number of rows.

You may drill down into Table Info to check if table schema is required to be adjusted for getting better SQL performance and space usage.
2. If the used size of a table/an index is far behind its allocated size and the table/index has no extra extent since it was created, you may consider to change the next extent size to avoid wasting space.
3. Even if extent pages become empty after you delete data, the disk space remains unavailable for use by other tables unless you reclaim the space. If you notice the used size of a table/an index is far behind its the allocated size and it has a large number of extents, you may consider to rebuild the table via ALTER INDEX, UNLOAD and LOAD, and ALTER FRAGMENT to release unused space within an extent to other tables/indexes.
