DBSpace Overview Panel

This page exhibits the number of page reads/writes performed on a specific DBSpace and the space utilization of the DBSpace in the period between Start and End, and lists active top busy I/O sessions sorted by the column RW+. The information will be helpful for diagnosing performance problems that are happening or had happened on the DBSpace and related to disk I/O.
 

Symptom and Approach

1. If the number of page writes that have occurred or are occurring on the DBSpace is high and the utilization of the DBSpace is almost saturated, you may need to find out what SQL activities are executing (or have been run) and what tables used by these SQLs have a large number of extent.
a. Right-click the point and drill down into SQL Point in Time to check what SQLs are running (or ran) and how SQLs are running (or ran) in the interval across that point in time.
b. If the table referred in SQLs at the point in time shows a large number of extents in Table Extent List (exclude system table), you may need to consider how to configure the table extent size and how to configure the DBSpace.
2. If the number of page reads that have occurred or are occurring on the DBSpace is large, you may need to find out what SQL activities are executing (or have been run) and what tables are used by these SQLs, what the busiest tables or indexes are and which tables may need to exercise Update Statistics.
a. Right-click the point and drill down into SQL Point in Time to check what SQLs are running (or ran) and how SQLs are running (or ran) in the interval across that point in time.
b. If tables referred in SQLs at the point in time are the busiest ones in Top Busy Table, you may need to optimize table schemas to reduce disk I/O issue.
c. If tables referred in SQLs at the point in time are found in Outdated Update Statistics Table or Incompleted Update Statistics Table, you may need to execute Update Statistics to reduce disk I/O issue.
3. For Top I/O Session - Real-Time, please refer to Top I/O Session to get an idea about how to catch and diagnose symptoms shown in the table.
 

How to Operate

 

When you click or right-click any place on a diagram, the red line shows as below.

The point-in-time values under the red line on diagrams are displayed in the Legend-Value table as below. They are listed in diagram order from top to bottom. The date and time of the red line shows above the Legend-Value table.

Here we briefly annotate legends appeared in the Legend-Value table.

  • "DBSpace Page Reads" is the legend for the line on the "DBSpace Page Reads" diagram.
  • "DBSpace Page Writes " is the legend for the line on the "DBSpace Page Writes" diagram.
  • "DBSpace Utilization" is the legend for the line on the "DBSpace Utilization" diagram.

If you would like to view trend charts over a long period of time, please click to pick the start and end dates separately or enter the start and end dates manually. Then click Refresh to redraw diagrams.
