OS CPU Panel

This page exhibits CPU busy rate and IO wait percentage in the period between the Start and the End, and lists active top busy processes sorted by the column CPU%. The information will be helpful for identifying and tracing performance issues that are happening or had happened on the monitored server.
 

Symptom and Approach

1. If the CPU is keeping a high utilization rate (exceeding 80%+) and the VB Busy% on the MainPanel is high, you may need to find out what SQL activities are executing and how the resource has been taken.
a. Right-click the high utilization rate on the diagram and drill down into SQL Point in Time to check what SQLs are running and how SQLs are running in the interval of the red line.
b. Use the Trend Analysis to view other trend charts for getting more information about virtual process, database disk activities, and virtual memory usage etc.
2. If the CPU IO Wait is keeping high (exceeding 10%+) as most of active processes are waiting for I/O operations to complete, you may need to identify which process cause this situation and disk with utilization rate exceeding 40%+.
a.

If the busiest process in the Top Busy OS Processes - Real-Time table or in the drilling-down Process List is a database process, oninit, you may need to check the Top I/O Session or the Top Busy Session to find out which session is running in the process through , how the resource has been taken by the session, and what SQLs are executing etc to get the root cause.

For Top Busy OS Processes - Real-Time, please refer to Top OS Process List to get an idea about how to catch and diagnose symptoms shown in the table.
b. Use the Trend Analysis to view trend charts for getting more information of disk activities.
 

How to Operate

   Trend Chart

 

When you click any place on the diagram, the red line shows as below.

The point-in-time values under the red line on diagrams are displayed in the Legend-Value table as below. They are listed in diagram order from top to bottom. The date and time of the red line shows above the Legend-Value table.

Here we briefly annotate legends appeared in the Legend-Value table.

  • "CPU Busy Rate: CPU(Avg)" and "CPU Busy Rate: CPU(Max)" are legends for lines representing the average CPU busy rate and the maximum CPU busy rate respectively on the "CPU Busy Rate(Max)/(Avg)" diagram.
  • "CPU Busy" is a legend prefix for each line on the "CPU Busy Rate" diagram. A line represents a CPU.
  • "CPU I/O Wait " is a legend prefix for each line on the "CPU I/O Wait" diagram. A line represents the I/O wait of a CPU.

If you would like to view trend charts over a long period of time, please click to pick the start and end dates separately or enter the start and end dates manually. Then click Refresh to redraw diagrams.

   Top Busy OS Processes - Real-Time

 

It is refreshed automatically every couple of seconds. You can click a column head to sort the list in ascending/descending order of the selected column. Please refer to Top OS Process List to get more information about how to diagnose symptoms shown in the table.
