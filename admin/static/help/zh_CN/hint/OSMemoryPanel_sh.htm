OS Memory Panel

This page exhibits the amount of memory paging in/out and the size of physical free memory in the period between the Start and the End, and lists top memory-consuming OS processes sorted by the column %MEM. The information will be helpful for identifying and tracing performance issues that are happening or had happened on the monitored server.
 

Symptom and Approach

1. As the amount of memory page out increases, CPU utilization also increases. If the amount of memory page out is keeping high and the size of free physical memory is small, you may need to check if the situation is caused by any database process, oninit. If it's caused by a database process, you need to find out what SQL activities has been run (or is running) and how the resource has been taken.
a. Right-click the point and drill down into SQL Point in Time to check what SQLs are/were running and how SQLs are/were running in the interval of the red line.
b. Use the Trend Analysis to view other trend charts for getting more information of resource usage/consumption.
2. If the most memory-consuming process in the OS Process Memory consumption - Real-Time table is a database process, oninit, you may need to check the Top Memory Consuming Session to find out which session is running in the process, how the resource has been taken by the session, and what SQLs are executing etc to get the root cause.
 

How to Operate

   Trend Chart

 

When you click any place on the diagram, the red line shows as below.

The point-in-time values under the red line on diagrams are displayed in the Legend-Value table as below. They are listed in diagram order from top to bottom. The date and time of the red line shows above the Legend-Value table.

Here we briefly annotate legends appeared in the Legend-Value table.

  • "OS Memory Page In" is the legend for the line on the "OS Memory Page In" diagram.
  • "OS Memory Page Out" is the legend for the line on the "OS Memory Page Out" diagram.
  • "OS Free Physical Memory" is the legend for the line on the "OS Free Physical Memory(MB)" diagram.

If you would like to view trend charts over a long period of time, please click to pick the start and end dates separately or enter the start and end dates manually. Then click Refresh to redraw diagrams.

   OS Process Memory consumption - Real-Time

 

It is refreshed automatically every couple of seconds to show current and active top most memory-consuming OS processes. From the COMMAND, you can identify whether the most memory-consuming process is a database process, oninit. You may need to check the Top Memory Consuming Session to find out which session is running in the most memory-consuming database process and get the root cause.

You can click a column head to sort the list in ascending/descending order of the selected column.
