Transaction Trend Panel

This page exhibits trend charts of the number of transactions, committed transactions and lock requests in the period between the Start and the End. These charts enable you to monitor the quantity of transactions and lock requests.

However, the database server incurs some overhead in obtaining a lock. For an operation that changes a large number of rows, obtaining one lock per row might not be cost effective. If the quantity of transactions is low but the number of lock requests is high, you may need to diagnose what kind of lock modes are used in the SQLs activities* executed in the period of time and how these SQLs have been executed to reduce overhead of lock request.
 

How to Operate :

   Trend Chart

 

As you click any place on the diagram, the red line shows as below.

The values under the red line for each diagram are displayed in the Legend-Value table as below. They are listed in diagram order from top to bottom. The date and time that the red line represents is shown above the Legend-Value table. For a diagram with more than one trend line representing separate cpu, device, DBSpace etc, the legend-value pair of each trend line will be displayed separately in the Legend-Value table with the same prefix.

Here we briefly annotate legends appeared in the Legend-Value table.

  • "DB Transactions" legends represent the number of database transactions at the point of time on the "DB Transactions" diagram.
  • "DB Commits" legend represents represent the number of committed transactions at the point of time on the "DB Commits" diagram.
  • "Lock Requests" legends represent the number of lock requests at the point of time on the "Lock Requests/sec" diagram.

If you would like to view trend charts over a long period of time, please click to pick the start and end dates separately or enter the start and end dates manually. Then click [Refresh] to redraw diagrams.
