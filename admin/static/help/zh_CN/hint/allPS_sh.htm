Process List

On this page, you will see information of system loading and all processes running on the system. The top busiest system processes are listed in descending order of the column %CPU. A running database process will show "oninit" in the COMMAND and running state in the status column, thus you are able to tell whether the busiest process is a running database process.

From the list, you are able to know who runs the process, how much time the process uses, what the current state of the process and how muchpercentage of CPU is occupied by the process etc.

You can click a column head to sort the list in ascending/descending order of the selected column and click or anytime to refresh information on the page. And you can navigate other pages of the list through the page navigator on the bottom of the page.

 

Symptom and Approach

If the busiest process in the table is a database process, oninit, and its CPU% is unreasonable high, you may need to use the Top I/O Session or Top Busy Session to find out which session is running in the process, how the resource has been taken by the session, and what SQLs are executing etc to get the root cause.
