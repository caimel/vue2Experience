Top I/O Session

The table shown on this page lists active sessions in the monitored database server* with extremenly busy I/O activities. They are listed in descending order of the column RW+. Values in the RW+, DskW+ and DskR+ are the incremental number of disk reads/writes activities exercised by a session in a refreshing interval.

Each session is displayed with how long it has been alive, how many reads/writes have been exercised up to now or in a refreshing interval, and what the database is been accessed etc.

You can click a column head to sort the list in ascending/descending order of the selected column and click or anytime to refresh information on the page.
 

Symptom and Approach

1.

If a session has tremendous amount of disk reads/writes in RW+ or DskR+, you may need to drill down to check what SQL is running, how the SQL is running, what tables are being accessed and how the resource is being taken by the SQL.

Related drilling-down options in the right-click menu : Active SQL Tree Diagram, List of Opened Table in a Session, SQL Cursor Held Info, Start Analysis Session, Active SQL in a Session
2. If a session has a large amount of disk reads and sequential scan in DskR+ and SeqScan respectively, you may need to drill down into Active SQL in a Session to analyze how the SQL is being executed and if any index is required.
3. If the DskW+ of a session is high, you may consider increasing database buffers.
4. If the busiest session is an abnormal one, you can terminate it with Terminate Session option in the right-click menu. This option will be shown only if you have permission to terminate a session.
