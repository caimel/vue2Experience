Top Busy Table

This page lists tables in descending order of the column Buf Reads, the number of buffer read operations. Each table is listed with its database, size, the number of rows in the table, the number of buffer read/write operations, the number of sequential scans, indication to tell whether it's a user-defined temporary table, sort file or permanent system temporary table, page lock setting indication, and the hit ratio of buffer read operations.

You can click a column head to sort the list in ascending/descending order of the selected column and click or anytime to refresh information on the page.


Column Description
Size(KB) Size of the table
N Rows The number of records in the table
Buf Reads The number of reads from the buffer pool and not from disk
Buf Writes The number of pages modified and sent back to the buffer pool
Re-Hit%

The percentage of pages found in the buffer without having to read them from the disk.
 

Symptom and Approach

1.

If a table has low buffer reads, you may drill down into Table Related SQL to check what SQLs are accessing the table and how the table is accessed by a SQL activity*, and drill down to Table Info to inspect the table schema design.
2. If a table has high sequential scans, you may take the following actions:
a. Drill down into Table Related SQL to check what SQLs are accessing the table and how the table is accessed by a SQL.
b. Drill down into Table Info to check if no proper index is set.
c. Check whether the statistics of the table is outdated via Outdated Update Statistics Table.
3. If Re-Hit% is low, you may need to increase buffers and restart the database server.
4. If the buffer writes is low, this could mean that no empty or unmodified buffer can be located, and you may need to allocate more buffer space. You may drill down into Table Related SQL to see what INSERT/UPDATE/DELETE SQLs are executing in the session and how they are executed.
