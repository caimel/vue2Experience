Outdated Update Statistics Table

The database server creates a statistical profile for all tables, indexes, and stored procedures when they are created. This statistical information is stored in system catalog tables and used by the optimizer to determine the lowest-cost execution plan of a SQL statement. These statistics frequently become outdated as the data changes and have to be updated using the UPDATE STATISTICS statement. However, the command imposes a further overhead on a busy system due to the large data sampling required for each table in a large database and can affect the performance.

This page exhibits tables that may require executing the UPDATE STATISTICS statement because of having had a number of rows inserted or deleted etc. You may analyze a table further to determine whether the UPDATE STATISTICS statement is required for the table. Each table listed in this page is shown with its database, the estimated and actual number of rows, percentage of rows changed, buffer read/write operations, and number of sequential scans.

You can click a column head to sort the list in ascending/descending order of the selected column and click or anytime to refresh information on the page.


Column Description
Changed% The number of rows inserted/deleted, expressed in a percentage value
Rows Estimated The estimative number of rows in the table
Actual Rows The actual number of rows in the table
Buffer Reads The number of reads from buffer for the table
Buffer Writes The number of writes to buffer for the table
Seqscans The number of sequential scans occurred on the table

   Criteria Window

 

You can specify a database that you would like to check. If a database is specified, only statistics information of tables in the selected database will be displayed; otherwise tables in each database on the database server will be displayed.
 

Symptom and Approach

If the Changed% of a table is extreme high (i.e. the table has executed lots of row insertion and/or deletion.), and its Buffer Reads is low or its Seqscans is high, you may take the following ways to determine if the UPDATE STATISTICS statement is required.

a. Drill down into Table Related SQL to check how the table contributes to a frequently-used or heavy-cost SQL activity
b. Drill down into Table Info to inspect the table schema design to see if any index/fragment is required before executing the UPDATE STATISTICS statement.

If the UPDATE STATISTICS statement is required for a table, you may drill down into Generate Update Statistics Script to generate the UPDATE STATISTICS statement for the table.
