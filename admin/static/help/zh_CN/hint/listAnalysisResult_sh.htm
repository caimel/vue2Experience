List Analysis Result

The page shows the analysis result of an analysis process which is not running, and a list of SQLs executed during the period of the analysis process. The list of executed SQLs is displayed in descending order of the column Process Time that represents how much time was taken to analyze the SQL. This page displays analysis job id, host name, status of the analysis job, the start and end time of the analysis process and the resource utilization of the session etc.

The table shows SQL activities executed in the analysis period. For each SQL activity, you are able to know what database it is executed on, whether sequential scan is used, what its cost is and how many rows are returned etc.

You can click a column head to sort the list in ascending/descending order of the selected column.
 

Symptom and Approach

1.

Cost is a measure of the resources and memory used by a SQL activity, calculated against the speed of the operation. More complicated a SQL activity is, higher cost it has.

If a SQL has high cost and uses sequential scan, you may need to drill down into SQL Evaluator via the right-click menu to check what the execution path is taken for this SQL activity to identify whether any database statistics is required or any index is required to be created or modified.

And you may also drill down into Execution History of SQL Pattern via the right-click menu to get more information like execution time for weighting whether or not consider it as a key point of performance tuning.
2. If a SQL returns a vast number of rows, application developer may need to check how the SQL statement is written and whether it is necessary to return such vast amount of data to avoid defective application design affecting database performance.
