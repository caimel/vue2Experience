Execution History of SQL Pattern

This page exhibits sessions that have executed the same SQL activity* in a specified period of time. It presents execution time, used database, cost and specifies whether sequential scan is exercised on the SQL activity for each session.

You can click a column head to sort the list in ascending/descending order of the selected column and click or anytime to refresh information on the page.
 

Symptom and Approach

The Cost is a measure of the resources and memory used by a SQL activity, calculated against the speed of the operation. More complicated a SQL activity is, higher cost it has.

1.

According to the execution time of a SQL, if it has high cost and uses sequential scan, you may need to drill down into SQL Evaluator from the right-click menu to check what the execution path was taken for this SQL activity to identify whether any index is required to be created or modified. And you may check whether the statistics of tables referred in the SQL are outdated via Outdated Update Statistics Table.
2. If a SQL returns a vast number of rows, application developer may need to check how the SQL statement is written and whether it is necessary to return such vast amount of data to avoid defective application design affecting database performance.
