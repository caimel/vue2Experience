CPU Bottleneck Panel

This page exhibits trend charts of the busy rate of CPU and database virtual processor, and the number of threads* in the ready queue in the period between Start and End, and lists active top busy sessions sorted by the column OP+. The information will be helpful for identifying and tracing performance bottlenecks that are happening or happened on the monitored server.
 

Symptom and Approach

1. If CPU is keeping a high utilization rate (exceeding 80%+) and database virtual process busy rate is close to the CPU rate during a period of time, you may need to find out the performance botttleneck comes from what SQL activities and how the resource has been taken by these SQLs.
a. Right-click the point and drill down into SQL Point in Time to check what SQLs are running (or ran) and how SQLs are running (or ran) in the interval across that point in time.
b. Use the Trend Analysis to view other trend charts for getting more information of system and database loading.
2. If the number of threads in ready queue is high, it indicates that CPU virtual processors are too busy to process all threads in the queue, you may need to find out the performance bottleneck comes from what heavy SQL activities and monitor the number of virtual processors.
a. Right-click the point and drill down into SQL Point in Time to check what SQLs are running (or ran) and how SQLs are running (or ran) in the interval across that point in time.
b. Use the Trend Analysis to view other trend charts related to CPU usage, disk activities, database transactions and database connections etc. to get the root cause.
3. For Top Busy Session - Real-Time, please refer toTop Busy Session to get an idea about how to catch and diagnose symptoms shown in the table.
 

How to Operate

 

There will be more than one line on the CPU Busy Rate and/or the DB VP Busy Rate diagram if there are more than one CPU and/or database virtual processor respectively on the monitored database server*. When you click any place on the diagram, the red line shows as below.

The point-in-time values under the red line on diagrams are displayed in the Legend-Value table as below. They are listed in diagram order from top to bottom. The date and time of the red line shows above the Legend-Value table.

Here we briefly annotate legends appeared in the Legend-Value table.

  • "CPU Busy" is a legend prefix for each line on the "CPU Busy Rate" diagram. A linerepresents a CPU.
  • "DB VP Busy Rate" is a legend prefix for each line on the "DB VP Busy Rate". A line represents a database virtual processor.
  • "VP Queue" is the legend for the line on the "VP Queue" diagram.

If you would like to view trend charts over a long period of time, please click to pick the start and end dates separately or enter the start and end dates manually. Then click Refresh to redraw diagrams.
