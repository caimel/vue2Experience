CPU Loading Panel

This page exhibits trend charts of CPU busy rate, database busy rate and system loading in the period between Start and End, and lists active top busy processes sorted by the column CPU%. The information will be helpful for identifying and tracing performance issues that are happening or happened on the monitored server.
 

Symptom and Approach

1. If CPU is keeping a high utilization rate (exceeding 80%+) and database virtual process busy rate is close to the CPU rate during a period of time, you may need to find out what SQL activities has been run and how the resource has been taken.
a. Right-click the point and drill down into SQL Point in Time to check what SQLs are running (or ran) and how SQLs are running (or ran) in the interval across that point in time.
b. Use the Trend Analysis to view other trend charts for getting more information of database loading.
2. OS loading represents the summary status of CPU, disk I/O, memory usage etc. If the OS average loading is high but CPU and database virtual process rates are low, you may need to identify which process is the bottleneck and how resources like disk and memory have been taken.
a. If the busiest process in the Top Busy OS Processes - Real-Time table is a database process, oninit, please refer to Top OS Process List to get an idea about how to catch and diagnose symptoms shown in the table.
b. Use Top I/O Session and Top Memory Consuming Session to find out which session has taken more resources and how SQLs are running in the session.
c. Use the Trend Analysis to view other trend charts related to disk activities for getting more idea about which resource has been taken.
 

How to Operate

 

There will be more than one line on the DB VP Busy Rate diagram if there are more than one database virtual processor on the monitored database server*. When you click any place on the diagram, the red line shows as below.

The point-in-time values under the red line on diagrams are displayed in the Legend-Value table as below. They are listed in diagram order from top to bottom. The date and time of the red line shows above the Legend-Value table.

Here we briefly annotate legends appeared in the Legend-Value table.

  • "CPU Busy Rate(Avg)" is a legend prefix for each line on the "CPU Busy Rate(Avg)" diagram.
  • "DB VP Busy Rate" is a legend prefix for each line on the "DB VP Busy Rate". A line represents a database virtual processor.
  • "OS Average Load" is the legend for the line on the "OS Average Load" diagram.

If you would like to view trend charts over a long period of time, please click to pick the start and end dates separately or enter the start and end dates manually. Then click Refresh to redraw diagrams.
