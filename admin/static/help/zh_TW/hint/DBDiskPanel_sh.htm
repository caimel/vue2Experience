Database Disk Panel

This page exhibits trend charts of disk I/O busy rate, the number of database page reads/writes and the number of sequential scans in the period between Start and End, and lists active top busy I/O sessions sorted by the column RW+. The information will be helpful for identifying and tracing performance issues that are happening or happened on the monitored server.
 

Symptom and Approach

1. If the disk utilization rate (exceeding 80%+) is high and the number of page writes that have occurred or are occurring on the database is high, you may need to find out what SQL activities are executing (or have been run) and how the resource has been taken.
a. Right-click the point and drill down into SQL Point in Time to check what SQLs are running (or ran) and how SQLs are running (or ran) in the interval across that point in time.
b. Use the Trend Analysis to view other trend charts for getting more information of database transactions and active SQLs.
2. If the number of page reads and sequential scans that have occurred or are occurring on the database is large, you may need to find out what SQL activities are executing (or have been run) and what tables are used by these SQLs, what the busiest tables are and what tables need to exercise Update Statistics.
a. Right-click the point and drill down into SQL Point in Time to check what SQLs are running (or ran) and how SQLs are running (or ran) in the interval across that point in time.
b. If tables referred by SQLs running at the point in time are the busiest ones in Top Busy Table, you may need to optimize table schemas to reduce disk I/O issue.
c. If tables referred by SQLs running at the point in time are found in Outdated Update Statistics Table or Incompleted Update Statistics Table, you may need to execute Update Statistics to reduce disk I/O issue.
3. For Top I/O Session - Real-Time, please refer to Top I/O Session to get an idea about how to catch and diagnose symptoms shown in the table.
 

How to Operate

 

There will be more than one line on theDisk I/O Busy Rate diagram if there are more than one disk device on the monitored database server*. When you click or right-click any place on a diagram, the red line shows as below.

The point-in-time values under the red line on diagrams are displayed in the Legend-Value table as below. They are listed in diagram order from top to bottom. The date and time of the red line shows above the Legend-Value table.

Here we briefly annotate legends appeared in the Legend-Value table.

  • "DB Page Reads" and "DB Page Writes" are legends for lines representing the amount of database page reads and the amount of database page writes respectively on the "DB Page Reads/Writes" diagram .
  • "Disk I/O Busy Rate" is a legend prefix for each line on the "Disk I/O Busy Rate" diagram. A line represents a disk device.
  • "DB Seqscans/sec" is the legend for the line on the "DB Seqscans/sec" diagram.

If you would like to view trend charts over a long period of time, please click to pick the start and end dates separately or enter the start and end dates manually. Then click Refresh to redraw diagrams.
