Database VP Panel

This page exhibits busy rate of database virtual processors of CPU class and the number of threads in the ready queue in the period between Start and End, and lists active top busy sessions sorted by the column OP+. The information will be helpful for identifying and tracing performance issues that are happening or had happened on the monitored server.
 

Symptom and Approach

1. A CPU virtual processor is a process scheduled by the OS to carry out all the CPU intensive processing. If the utilization rate of CPU virtual processors is high, you may need to find out what SQL activities are executing.
a. Right-click the high utilization rate on DB VP Busy Rate and drill down into SQL Point in Time to check what SQLs are/were running and how SQLs are/were running in the interval across that point in time.
b. Use the Trend Analysis to view other trend charts related to CPU usage, disk activities, database transactions and database connections etc. to get the root cause.
2. If the number of threads in ready queue is high, it indicates that CPU virtual processors are too busy to process all threads in the queue. You may need to find out what heavy SQL activities are executing in the given period of time and monitor the number of virtual processors.
a. Right-click the point on a line and drill down into SQL Point in Time to check what SQLs are/were running and how SQLs are/were running in the interval across that point in time.
b. Use the Trend Analysis to view other trend charts related to CPU usage, disk activities, database transactions and database connections etc. to get the root cause.
3. For Top Busy Session - Real-Time, please refer to Top Busy Session to get an idea about how to catch and diagnose symptoms shown in the table.
 

How to Operate

 

There will be more than one line on the DB VP Busy Rate if there are more than one database virtual processor on the monitored database server*. When you click any place on the diagram, the red line shows as below.

The point-in-time values under the red line on diagrams are displayed in the Legend-Value table as below. They are listed in diagram order from top to bottom. The date and time of the red line shows above the Legend-Value table.

Here we briefly annotate legends appeared in the Legend-Value table.

  • "DB VP Busy Rate " is a legend prefix for each line on the "DB VP Busy Rate" diagram. A line represents a database virtual processor.
  • "DB VP Busy Rate(Avg) " is the legend for the line on the "DB VP Busy Rate (Avg) ".
  • "VP Queue" is the legend for the line on the "VP Queue" diagram.

If you would like to view trend charts over a long period of time, please click to pick the start and end dates separately or enter the start and end dates manually. Then click Refresh to redraw diagrams.
