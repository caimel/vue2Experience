Top Consumption Panel

This page gives you an overview of what sessions use much more CPU resources, have heavy disk I/O activities and consume lots of memory. Monitoring information in these lists that are refreshed every couple of seconds automatically, you can identify which session have heavy CPU, disk I/O and memory loading and drill down the session to find out the root cause.

The Top Busy Sessions shows information of active top busy sessions running in the database server. They are listed in descending order of the [OP+] column. [OP+] column, the incremental operations like read/write and networking etc., and [SEQ+] column, the incremental sequential scans, are the number of activities exercised by a session in a refreshing interval. Each session is displayed with how long it has been alive, how many operations have been executed up to now or in a refreshing interval, how many sequential scans have been exercised up to now or in a refreshing interval, and what the database is being accessed etc. Please refer to Top Busy Sessions for more information of symptoms and approaches found in the tabular list.

The Top I/O Sessions shows information of active sessions in the database server with extremenly busy I/O activities. They are listed in descending order of the [RW+] column. [RW+], [DskW+] and [DskR+] columns are the incremental number of disk reads/writes activities exercised by a session in a refreshing interval. Each session is displayed with how long it has been alive, how many reads/writes have been exercised up to now or in a refreshing interval, and what the database is been accessed etc. Please refer to Top I/O Sessions for more information of symptoms and approaches found in the tabular list.

The Top Memory Sessions shows information of active sessions in the database server that consume the most memory. They are listed in descending order of the [Mem Allocated] column. Each session is displayed with how long it has been alive, how many memory have been allocated and used, how many locks are held by the session, how files are opened in a session, and how many buffer reads/writes have been exercised up to now etc. Please refer to Top Memory Sessions for more information of symptoms and approaches found in the tabular list.
 

Symptom and Approach :

1.

If a session in the Top Busy Sessions has tremendous amount of [OP+] or [SEQ+], you may need to drill down to check what SQL is running, how the SQL is running and how the resource is being taken.
2.

If a session in the Top I/O Sessions has tremendous amount of [RW+] or [DskR+], you may need to drill down to check what SQL is running, how the SQL is running, what tables are being accessed and how the resource is being taken by the SQL.
3. If a session in the Top I/O Sessions has a large amount of [DskR+] and [Sequence Scans], you may need to drill down to Active SQL in a Session for analyzing how the SQL is being executed and if any index is required.
4. If the [DskW+] of a session in the Top I/O Sessions is high, you may consider increasing database buffers .
5.

If a session in the Top Memory Sessions has tremendous amount of [Mem Allocated], you may need to drill down to Active SQL Tree Diagram, SQL History for a Session, Start Analysis Session and Active SQL in a Session of the memory-consuming session via right-click to check what SQL has been executed (is executing) and how the resource is being taken to get the root cause.
6. If the top most memory-consuming session in the Top Memory Sessions has a tremendous value shown in [File Opened] column, you may need to drill down to SQL Cursor Held Info via right-click to find out which SQLs haven't freed their cursors.
7. If the top buiest sessions in these three tables are the same, you may consider analyzing the session via Start Analysis Session to check what SQLs have been executed, the cost of each SQL activity * and how they have been executed for SQL tuning.
8. If the busiest session is an abnormal one, you can terminate it with [Terminate Session] in the right-clicking context menu. This context menu item will be shown only if you have permission to terminate a session.
 

How to Operate :

 

You can click on a column head of any table to sort the table in ascending order of the column. Clicking on the same column head will sort the table in descending order of the column.
