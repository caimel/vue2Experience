Active Connections

This page exhibits active sessions in the monitored database server* in ascending order of the connection idle time (ie. the column Idle Time). For each session, you will know when the connection is established, the number of locks that are held or waiting in the connection, and the number of read/write operations from/to buffer cache etc.

You can click a column head to sort the list in ascending/descending order of the selected column and click or anytime to refresh information on the page.


Column Description
Connect Time When a database connection is established.
Idle Time The duration of connection that has been idle
Locks Held

The number of locks held by session

Lock Wait The number of locks that the session is waiting
Buffer Read The number of reads from the buffer pool, not from disk. Higher the buffer reads is, higher the chance of finding data from the buffer pool is.
Buffer Write The number of pages modified and sent back to the buffer pool
 

Symptom and Approach

1. If the buffer reads of a session is low, you may drill down Active SQL Tree Diagram, SQL Cursor Held Info, Start Analysis Session and Active SQL in a Session to check what SQLs are running, how a SQL activity* is executed and how the resource has been taken by a SQL activity to get the root cause.
2. If the buffer writes is low, this could mean that no empty or unmodified buffer can be located, and you may need to allocate more buffer space. You may drill down Start Analysis Session and SQL History of Session to see what INSERT/UPDATE/DELETE SQLs are executing in the session and how they are executed.
3. If you notice the number of connections surging above normal values, then you may want to terminate idle connections that have been sitting idle for long durations in order to release the resources. You can select Terminate Session option in the right-clicking menu to terminate a session. This option will be shown only if you have permission to terminate a session.
