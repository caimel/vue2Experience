Top Running SQLs

This page lists top cost running SQLs that currently are being executed on the monitored database server*. They are displayed in descending order of the column Cost. For each SQL activity*, it tells you how many rows are returned, which database the activity is on, how complicated the SQL activity is etc.

You can click a column head to sort the list in ascending/descending order of the selected column and click or anytime to refresh information on the page.
 

Symptom and Approach

The Cost is a measure of the resources and memory used by a SQL activity, calculated against the speed of the operation. More complicated a SQL activity is, higher cost it has.

1.

If a SQL has high cost, you may need to drill down into SQL Evaluator, Active SQL Tree Diagram, Start Analysis Session or Execution History of SQL Pattern from the right-click menu to check what execution path is taken by this SQL activity* and how resources are consumed by the SQL activity to identify whether any database statistics is required or any index is required to be created or modified.
2. If a SQL returns a vast number of rows, application developer may need to check how the SQL statement is written and whether it is necessary to return such vast amount of data to avoid defective application design affecting database performance.
