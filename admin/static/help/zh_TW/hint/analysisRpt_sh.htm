Performance Analysis Report

This page displays a comprehensive report of the entire database system operated in the specified period of time. This report provides graphical trend charts of OS and database characteristics, tabulated high-frequency or cost-intensive SQL statements, incomplete and/or outdated table statistics, and expert tuning guidelines based on an in-depth analysis of the database system. And the performance report offers advice and suggestions about using the UPDATE STATISTICS command, optimizing the database design for speed, and improving the operating system and CPU performance. This report makes your job as a database administrator much easier, as you no longer have to spend huge amounts of time obtaining diagnostic information from the databases and you can use the suggestions offered by the report to instantly streamline the performance of the system.
 

Scenarios

   Outdated table statistics

 

The Informix query optimizer depends on the system catalog tables to decide the SQL execution path. Thus, if the system catalog data is out-of-date, the optimizer may generate incorrect and inefficient execution paths.

In such a scenario, you need to locate database tables that have outdated statistics and are involved in a majority of database activity. Then, you can run the UPDATE STATISTICS command for these tables to improve performance.

The database can obtain benefit from running the UPDATE STATISTICS command on these tables that have low or zero value of NRows in Update Statistics and big value of Accurated NRows in Tables with Outdated UPDATE STATISTICS Information, and are the top busiest in Tables with Outdated UPDATE STATISTICS Information.

   Incorrect Indexing

 

The database server depends on indexes for table columns to access data from large tables. If a column does not have an index specified, the server uses basic sequential scanning to access table data. This technique can be really costly for the server to access very large database tables that have a very low hit rate for restricted query criteria. Using an index for frequently accessed data columns is thus an effective data retrieval technique.

In such a scenario, you need to identify SQL statements that are used frequently in SQL Statements with High Cost and have high cost in SQL Statements with High Frequency, and then create indexes for the columns that are being frequently accessed.
 

How to Operate

   Items and data criteria for the analysis report

 

You must specify a range of date and time for generating the analysis report and the items to be exhibited on the report. All items are classified into individual group. You can choose item one by one from the complete list or from its own group. Any item listed under Choose Item will be shown on the report.

You can click or to select an item or all items respectively into Choose Item, and click or to remove an item or all items respectively out of Choose Item. Then click Submit to generate the report. You can click to print out the report.
