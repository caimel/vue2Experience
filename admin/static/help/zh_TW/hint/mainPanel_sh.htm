MainPanel

This page gives an overview of current performance status of the monitored database server*. It includes several sections to show OS and DB level KPIs, top busy sessions and message log of the monitored database server respectively.

As high CPU load, high disk I/O, frequency memory swap or unusual busy session shows, you may drill down from the corresponding KPI chart or the suspicious session to identify that performance bottlenecks come from OS or database level.
 

Symptom and Approach

   OS Overview

 
  CPU Busy %
   

The KPI chart shows variations in CPU utilization rate. If CPU is keeping a high utilization rate (exceeding 80%+), you may consider that CPU has became the bottleneck. You may need to find out how the resource has been taken and which process has taken the resource in order to find any possible way to reduce the CPU utilization.

Related drilling-down options in the right-click menu : OS CPU Panel, CPU Loading Panel, CPU Bottleneck Panel, Top OS Process List
  Disk Busy %
   

The KPI chart shows variations in disk I/O rate. If disk is keeping busy (exceeding 80%+), you may need to find out how the resource has been taken and which session has taken the resource in order to find any possible way to reduce the disk I/O.

Related drilling-down options in the right-click menu : OS Disk Panel, Disk Loading Panel, Disk Bottleneck Panel
  Page In Out / Sec
   

The KPI chart shows variations in memory page swap. If the amount of page swapped in/out is keeping high, it definitely will hurt performance a lot; you may need to find out how the resource has been taken and which session has taken the resource in order to find any possible way to reduce the memory swap.

Related drilling-down options in the right-click menu : OS Memory Panel, Memory Loading Panel, Memory Bottleneck Panel
  Host*
   

It gives an idea of system loading. If any unusual situation is observed, you may diagnose what the running processes are, how the resource has been taken and the network connection status in order to find any possible way to solve the unusual situation.

Related drilling-down options in the right-click menu : Top OS Process List, Process List, All Network Connections/Network Connections in Waiting

   Database Overview

 
  VP Busy %
   

The KPI chart shows the CPU utilization rate of database server virtual processes. If it is keeping high and close to the rate on the CPU Busy %, you may need to check the virtual process queue, the virtual process loading and the CPU loading, and find out which session has taken the resource in order to find any possible way to reduce the CPU utilization.

Related drilling-down options in the right-click menu : Database VP Panel, Top Busy Session, Top Running SQL
  DB Disk RW / Sec
   

The KPI chart shows variations in the size of disk read/write on the database server. If disk read/write size is large and the Disk Busy % shows a very busy rate, you many need to check the disk loading and find out which session has taken the resource in order to find any possible way to reduce the disk I/O rate.

Related drilling-down options in the right-click menu : Database Disk Panel, Top I/O Session
  Virtual Mem Alloc / Used
   

The KPI chart shows variations in the size of allocated and used virtual memory. The used size depends primarily on the types of applications and queries that you are running. The size of the free virtual memory will affect how busy disk I/O will be and how many sessions can be run simutaneously. If the size of the free virtual memory is reducing, you many need to check the utilization of memory pool, the OS memory loading and the database memory loading, and find out which session has taken the resource in order to find any possible way to reduce the used virtual memory.

Related drilling-down options in the right-click menu : Database Memory Trend Panel, DB System Memory Pool, Top Memory Consuming Sessions

  DB Server
   

It gives an idea of database server loading. If any unusual situation is observed, you may check what the running sessions are, how the resource has been taken by the top sessions or top SQLs in order to find any possible way to solve the unusual situation.

Related drilling-down on the right-click menu : Database Server Status, Database Information, Active Connections , Top Busy Session, Top Running SQL

   Top Busy Session

 
 

The table lists active busiest sessions on the database server. It is sorted based on the column OP+ and will be refreshed automatically every couple of seconds. The OP+, and the SEQ+ are the incremental operations like read/write and networking etc. and the incremental sequential scans exercised by a session respectively in a refreshing interval. You may need to drill down the session with tremendous amount in the OP+ or the SEQ+ to check what SQLs are running in the session, how SQLs in the session are running and how the resource has been taken by SQLs in the session.

You can click a column head to sort the list in ascending/descending order of the selected column.

Related drilling-down options in the right-click menu : Active SQL Tree Diagram, Active SQL in a Session, Start Analysis Session

The login user with permission to terminate session can terminate an abnormal session by selecting Terminate Session option in the right-clicking context menu.

   Message Log

 
 

It lists the most latest database message logs. You may drill down to view more database message logs.

Related drilling-down options in the right-click menu : Outstanding Database Message Log, Database Message Log
