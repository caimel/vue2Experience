Top Memory Consuming Session

This page lists the top most memory-consuming sessions on the database server* in descending order of the column Mem Allocated. For each session, you are able to know how long the session has been alive, how many memory has been allocated, how many files have been opened and what the database the session is accessing etc.

You can click a column head to sort the list in ascending/descending order of the selected column and click or anytime to refresh information on the page.
 

Symptom and Approach

The File Opened represents the number of files opened in a session and relates to the memory size usage and unfreed cursors.

1.

If a session has tremendous amount in the Mem Allocated, you may need to drill down into Active SQL Tree Diagram, Start Analysis Session and Active SQL in a Session of the memory-consuming session to check what SQL has been executed (is executing) and how the resource is being taken to get the root cause.
2. If the top most memory-consuming session has a tremendous value in the File Opened, you may need to drill down into SQL Cursor Held Info to find out which SQLs haven't freed their cursors.
3. You may drill down into Top Busy Session to check if those most memory-consuming sessions are top busiest sessions. Please refer to Top Busy Session for more information of symptoms and approaches about top busiest sessions.
4. If the session is an abnormal one, you can terminate it with Terminate Session option in the right-click menu. This option will be shown only if you have permission to terminate a session.
